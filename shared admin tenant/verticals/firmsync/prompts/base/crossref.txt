CROSS-REFERENCE CHECK MODULE:
If Cross-Reference Check is enabled, verify internal document references:

1. INTERNAL REFERENCE SCANNING:
   - Scan for all internal references (e.g., "see Section 5.2") and verify they exist
   - Locate section numbers, clause references, defined terms
   - Identify references to exhibits, schedules, or attachments

2. REFERENCE VALIDATION:
   - Flag any broken, undefined, or circular references
   - Verify that referenced sections actually exist
   - Ensure defined terms are used consistently throughout the document

3. EXTERNAL REFERENCE REVIEW (Optional):
   - Note if external references (e.g., statutes, case law) are cited but unclear or incomplete
   - Flag incomplete or ambiguous external citations

4. ISSUE IDENTIFICATION:
   - Flag broken references with specific location citations
   - Identify undefined terms or inconsistent usage
   - Note circular or self-referential problems

5. SUGGESTED CORRECTIONS:
   - Suggest specific fixes where possible
   - Do not fabricate section names or definitions
   - Use professional tone: "Section 6.1 refers to undefined term 'Preferred Vendor'. Consider defining this earlier."

6. STATUS CLASSIFICATION:
   - Valid: Reference exists and is accurate
   - Invalid: Reference points to non-existent section or contains errors
   - Missing: Referenced item should exist but is absent
   - Undefined: Term used but not properly defined

7. IMPROVEMENT RECOMMENDATIONS:
   - Use measured language for all suggestions
   - Provide specific location references for all issues
   - Flag significant inconsistencies for attorney review

Output as JSON with reference locations, status, and suggested corrections.
